                Joe Sneddon Moderator benjaoming • a year ago

                I install the packages myself and look them over. I don't promote/feature/recommend PPAs that I wouldn't use myself, and if it's a "kitchen sink" PPAs that has about a gazillion packages in, I usually give a warning.

                The packages in this PPA seem to have been built by someone who was involved in the bug report discussions etc, which, for me at least, suggests they're not some fly-by-night malware spreader.

                But yes, there are inherent security risks with third-party PPAs and repositories -- and users should always double-check things themselves. That said, I don't think anyone has ever actually acquired or repurposed a PPA and used it to spread malware. But, yeah, it's always possible. It's why new technologies like Snap and Flatpak have emerged.
